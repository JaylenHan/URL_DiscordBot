# 회고 & 로드맵 (Retrospective & Roadmap)

## 깨달은 점 (Lessons Learned)

- 모델을 연동한 디스코드 봇을 구현하며 **실질적 서비스 제공의 가능성**을 확인했다.
- 일반적으로 **깨끗하고 방대한 URL 데이터를 구하기 쉽지 않다**.
- 성능을 위해서는 **효과적이고 획기적인 Feature**가 필요하다.
- 현실의 URL은 상상 이상으로 다양한 패턴(정상/악성)이 존재해 **오탐과 미탐은 필연적**이다.
- AI는 **지속적인 학습과 개선**이 필요하다.

## 기술적 회고

| 결정 | 트레이드오프 | 평가 |
|---|---|---|
| RF + WHOIS 이중 검증 | 정확도 ↑ ↔ 응답 지연(외부 조회) | 오탐 보완에 효과적 |
| 난독화 정규식 파싱 | 우회 대응 ↑ ↔ 룰 유지보수 | 실제 우회 패턴 대응에 필요 |
| 원격 모델 fetch | 코드/모델 분리 ↔ 자격증명 관리 | 시크릿 관리 주의 필요(보안 이슈 발생) |
| Heroku worker 배포 | 상시 구동 단순 | 의도대로 동작 |

## 보안 후속 조치 (필수)

- 하드코딩된 모델 fetch 자격증명 **폐기 + 환경변수 전환** (→ [시작 가이드 > 보안 주의](../03-guides/GETTING_STARTED.md#보안-주의))

## 향후 방향성 (Roadmap)

- **보안 기능 확장** — URL 외 악성 파일·링크, 위장 Nitro 코드 생성기 등 신종 위협 대응
- **보안 커뮤니티 협력** — 최신 위협 정보 공유 네트워크 구축, 모델·기능 지속 업데이트
- **사용자 교육·인식 제고** — 경고 시 구체적 예방 조치·안전 사용법 안내 제공
